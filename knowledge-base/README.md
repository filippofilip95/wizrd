# Knowledge Base

This folder contains compounding intelligence - everything you learn becomes reusable.

## Folder Structure

```
knowledge-base/
├── playbooks/               # How-to guides and patterns
│   └── company-as-code-foundation.md
│
├── lessons-learned/         # Post-project insights (create as needed)
│
└── industry-research/       # Domain knowledge (create as needed)
```

## Knowledge Compounding Loop

```
1. Complete a project
   ↓
2. Extract lessons → /lessons-learned/
   ↓
3. Generate content from lessons
   ↓
4. Update industry research → /industry-research/
   ↓
5. Improve playbooks → /playbooks/
   ↓
6. Next project starts smarter
```

## Creating Lessons Learned

After each significant project:

```markdown
# [Project Name] - Lessons Learned

## What Worked
- [Success 1 with metrics]
- [Success 2 with metrics]

## What Didn't Work
- [Failure 1 with context]
- [Failure 2 with context]

## Key Insights
- [Insight that applies to future projects]

## Reusable Patterns
- [Pattern that can be templated]

## Recommendations
- For similar projects: [advice]
- For the client type: [advice]
```
